ШПАРГАЛКА ПО КУРСОВОЙ РАБОТЕ
Тема: «Разработка интернет-магазина на основе HTML, CSS и JavaScript»
Автор: Липатов Евгений Сергеевич, 2 курс, ИСиП 1 группа
Преподаватель: Матыцин В.С.


1. ЧТО СДЕЛАНО?
Разработан прототип интернет-магазина электроники «ТехноМир» на чистом HTML, CSS, JS без фреймворков (SPA). Есть каталог с фильтрацией, корзина, сравнение до 4 товаров, детальные карточки.

2. ЦЕЛЬ РАБОТЫ
Спроектировать и реализовать функциональный прототип интернет-магазина.

3. ЗАДАЧИ (6 штук, легко запомнить)
- Изучить теорию e-commerce
- Реализовать каталог с фильтрацией (категория, бренд, год)
- Реализовать корзину (подсчёт суммы, управление)
- Реализовать модуль сравнения (до 4 устройств)
- Подключить фото с CDN производителей
- Предложить подключение API (Fetch + REST)

4. ОБЪЕКТ И ПРЕДМЕТ
Объект: интернет-магазин как информационная система.
Предмет: методы применения HTML, CSS, JS.

5. СТРУКТУРА КУРСОВОЙ (3 главы + введение/заключение)
- Глава 1: теория (интернет-магазин как ИС, HTML, CSS, JS, Fetch API и REST)
- Глава 2: практика (описание проекта, HTML-разметка, CSS, JS-логика, тестирование)
- Глава 3: по факту нет отдельной главы 3, но в файле есть разделы 2.5 тестирование, вывод, заключение. Будьте готовы: после главы 2 сразу «ВЫВОД» и «ЗАКЛЮЧЕНИЕ».

6. ТЕХНОЛОГИИ (ключевые моменты)
- HTML5: семантические теги, атрибуты data-*, alt, onerror для изображений.
- CSS3: переменные (custom properties), Grid (repeat(auto-fill, minmax(300px, 1fr))), Flexbox, анимации (cubic-bezier), glassmorphism (backdrop-filter: blur).
- JavaScript: SPA, управление состоянием (state store), делегирование событий (closest), динамическая отрисовка DOM.

7. АРХИТЕКТУРА SPA vs MPA
SPA: одна загрузка HTML, динамическая навигация, быстрее, сложнее разработка.
MPA: перезагрузка при каждом клике, медленнее, проще SEO.

8. КАТАЛОГ И ФИЛЬТРАЦИЯ
18 моделей трёх брендов (Apple, Samsung, Xiaomi). Фильтры по категории (смартфоны/ноутбуки/аудио), бренду, году. Реализовано через PRODUCTS.filter().

9. КОРЗИНА
Объект cart: ключ "id_хранилище", значение — количество. Сумма через reduce. Удаление, изменение количества. Анимация значка .pop.

10. СРАВНЕНИЕ
Массив compareList (макс 4). Toggle-логика. Динамическая таблица характеристик: Set для уникальных полей, flatMap, find.

11. CSS-ДИЗАЙН-СИСТЕМА
Тёмная тема (dark premium). Переменные: --bg, --acc, --card и др. Glassmorphism в навигации. Адаптивная сетка без медиа-запросов.

12. FETCH API И REST API (отдельный раздел 1.5)
Fetch API — браузерный интерфейс для HTTP-запросов, основан на Promise.
REST — архитектурный стиль: ресурсы по URL, методы GET/POST/PUT/DELETE.
В прототипе данные статические, но легко заменить на fetch('/api/products').

13. ТЕСТИРОВАНИЕ (раздел 2.5)
Использовались Chrome DevTools (Console, Elements, Network, Device Toolbar). Проверены: фильтрация, корзина, сравнение, адаптивность (375px, 768px, 1280px). Обработчик onerror при загрузке фото — прозрачность 8%.

14. ПРАВОВЫЕ АСПЕКТЫ (важно для защиты!)
Не на все фото с CDN производителей действуют единые правила. В учебном проекте использование допустимо, но в коммерции нужны собственные фото или лицензии. В работе есть примечание об этом.

15. ВЫВОДЫ И ЗАКЛЮЧЕНИЕ
- SPA на чистом JS работает.
- CSS-переменные упрощают поддержку.
- Grid обеспечивает адаптивность.
- Делегирование событий экономит память.
- CDN-фото — удобно, но нужна осторожность с правами.

16. ЧТО МОГУТ СПРОСИТЬ НА ЗАЩИТЕ?
- Почему выбрали SPA? (быстрая навигация, нет перезагрузок)
- Как храните данные о товарах? (массив объектов PRODUCTS)
- Как фильтруете? (метод filter, условия по категории/бренду/году)
- Как работает корзина? (объект cart, ключ id+storage, сумма через reduce)
- Как ограничили сравнение 4 товарами? (условие в toggleCompare: if compareList.length < 4)
- Что такое делегирование событий? (один обработчик на родителе, определяем элемент через closest)
- Какие CSS-фишки использовали? (переменные, Grid, glassmorphism, анимации)
- Как обеспечили адаптивность? (repeat(auto-fill, minmax(300px,1fr)) и медиа-запрос для детальной страницы)
- Что такое Fetch API? (встроенный браузерный метод для запросов к серверу)
- Почему нельзя просто взять фото с сайта Apple? (авторские права, товарные знаки)
- Как бы вы подключили бэкенд? (заменить PRODUCTS на fetch, добавить отправку корзины POST-запросом)
